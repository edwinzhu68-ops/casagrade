# 上线前自测清单

部署到服务器后，按下列项逐条自测，打勾表示通过。

---

## 一、环境与配置

- [ ] 后端已启动（`PORT` 或默认 3000），无报错
- [ ] 前端各页 `API_BASE` 已改为实际 API 地址（如 `https://api.casagrade.com`；index 支持 localStorage `lottery_api_base` 或自动用线上/本地）
- [ ] 浏览器能正常访问前端页面与后端接口（无 CORS 报错）
- [ ] 若前端与 API 不同域，后端 CORS 已放行该域名（`main.ts` 用 `ALLOWED_ORIGINS` 或默认含 www.casagrade.com / api.casagrade.com）
- [ ] 总后台登录框无默认账号/密码（已移除占位值，生产环境务必改管理员密码）

---

## 二、顾客端（index.html）

- [ ] 输入 5 位店号可进入店铺，展示 Billete/Chance 与单价
- [ ] 九宫格选号、数量，点「确认投注」生成订单，出现订单号、金额、二维码、30 分钟倒计时
- [ ] 待付款页二维码可供老板端输入订单号或粘贴二维码内容后查询（PC 无扫码，直接输入订单号）
- [ ] 老板确认收款后，顾客端自动跳转到「完成付款」页（或轮询几秒内更新）
- [ ] 首页「已付款订单」列表展示多笔订单（再下一单并付款后，列表有多条）
- [ ] 点某笔已付款订单可进入详情，展示订单号、金额、中奖/未中奖/已兑奖状态与二维码
- [ ] 开奖后：未中奖显示「未中奖」，中奖显示金额与领奖提示，已兑奖显示「已兑奖」
- [ ] 点击「Salir」退出店铺返回首页，再次进入同一店号可正常下单（账号/订单列表仍保留）

---

## 三、老板端（merchant.html）

- [ ] 登录页：账号/店号 + 密码可登录；勾选「不点退出则一直保持登录」后关闭再打开仍为已登录
- [ ] 登录后收银台展示店号、今日销售、待确认订单、等待兑奖（若有时）
- [ ] **待确认订单**：新订单约 3 秒内出现；点「确认」后该单消失，顾客端变为已付款
- [ ] **等待兑奖**：输入订单号后 4 位点「完成兑奖」—— 若唯一则直接弹兑奖确认；若多条则列出可选，选一条后确认兑奖
- [ ] **输入订单号**：收银台点「输入订单号」弹窗，输入完整订单号或后 4 位（或粘贴手机扫到的二维码内容）点「查询」—— 未付款可确认收款，已中奖可兑奖
- [ ] 若输入的订单不属于本店，提示「此订单不是在此店买的」
- [ ] 列表内某单直接点「完成兑奖」可弹出确认并完成兑奖
- [ ] 左上角 🇵🇦 点击可切换中文/西班牙语，刷新后语言保持
- [ ] 点击「退出」后 token 清除，再次进入需重新登录

---

## 四、开奖结算页（result.html）

- [ ] 从收银台点「开奖结算」跳转后，展示本店店号与开奖/销售/赔付/利润
- [ ] 有开奖结果时显示 1/2/3 奖号码；无开奖时显示「等待开奖」与倒计时
- [ ] 今日销售额、Chance/Billete 销售、总赔付、今日利润、销售明细、中奖名单等数据正确
- [ ] 最近 7 期结算记录有数据（若已有历史）
- [ ] 左上角 🇵🇦 切换语言与收银台一致并保持

---

## 五、总后台（dashboard.html）

- [ ] 管理员登录后可见手动开奖、发送开奖时间、店铺销售明细、全部店铺总销售、店铺对比等
- [ ] 选择日期时间、填写 1/2/3 奖号码后「发送开奖结果」，各端能收到开奖并更新中奖状态
- [ ] 「发送开奖时间」后，顾客端/老板端倒计时与设定一致（如有）

---

## 六、数据与防错

- [ ] 订单 30 分钟未付款后，后端将该单标为取消，老板端待确认列表不再显示
- [ ] 同一中奖单兑奖一次后，再次扫同一二维码或再兑一次会提示已兑奖
- [ ] 顾客端多笔已付款订单（至少 2 笔）均能正确展示并点进详情

---

## 七、可选抽查

- [ ] 顾客下单后，后端订单表 `ip_address` 为真实 IP（直连时为客户端 IP，经 Nginx 等代理时为 `X-Forwarded-For` 首段）
- [ ] 前端 `DEBUG = false` 时，控制台无业务调试日志（merchant/result 已用 `if (DEBUG) console.log`，上线保持 `DEBUG = false`）
- [ ] 后端生产环境已设置 `ADMIN_TOKEN`（总后台开奖/结算等接口需此头；未设置时 guard 不校验，仅适合本地开发）

---

**说明**：若某项为「暂无数据」或「未到开奖时间」导致无法验证，可标注「待有数据后再测」。全部通过即可视为具备上线测试条件。
