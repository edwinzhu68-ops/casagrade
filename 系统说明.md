# 彩票系统 — 总说明与功能

## 一、系统是什么

这是一套**巴拿马风格彩票销售与兑奖系统**，包含：
- **顾客端**：选号、下单、出示二维码付款/兑奖
- **老板端（收银台）**：确认收款、查看销售、扫码兑奖
- **总后台**：设置开奖时间、发送开奖结果、查看全平台数据

顾客不用注册，输入店号即可下单；老板需要登录自己的收银台

---

## 二、三种角色与入口

| 角色     | 入口页面        | 说明                     |
|----------|-----------------|--------------------------|
| 顾客     | `index.html`    | 输入 5 位店号进入店铺下单 |
| 老板     | `merchant.html` | 登录后进入收银台         |
| 结算查看 | `result.html`   | 老板从收银台点「开奖结算」进入 |

---

## 三、顾客端功能（简单易懂）

- **选店**：输入 5 位店号（如 88888）进入该店。
- **选号**：选 Billete（4 位数）或 Chance（2 位数），用九宫格输入号码和数量。
- **下单**：点「确认投注」后进入**待付款页**：
  - 显示订单号、购买的号码与金额、**总金额**、**30 分钟倒计时**；
  - 显示**二维码**，顾客到店给老板扫此码付现金。
- **付款**：老板在收银台点「确认收款」后，顾客端自动跳到**完成付款页**（等待开奖）。
- **开奖后**：
  - **未中奖**：显示「未中奖」；
  - **中奖**：显示中奖金额与「凭此单/二维码到店领奖」；
  - **已兑奖**：显示「已兑奖」。
- **订单入口**：首页下方有「已付款订单」入口，可随时点进去看订单详情、二维码与中奖/兑奖状态（二维码由页面实时生成，无需截图保存；换设备时若支持「订单号+店号」查询也可再次打开并出示）。

---

## 四、老板端功能（简单易懂）

- **登录/注册**：账号+密码，可勾选「保存登录信息」；支持用账号或店号登录。
- **待确认订单**：未付款订单列表，点「确认」即确认收款，顾客端会立刻变为已付款。
- **等待兑奖**：开奖后，已中奖且未兑奖的订单会列在「等待兑奖」。老板可：**扫码**顾客中奖二维码、在列表中**选中某单点「完成兑奖」**、或**输入订单号**后点「完成兑奖」。二维码内包含店号，若扫到的订单店号与当前店铺不一致，会提示「此订单不是在此店买的」。
- **销售数据**：今日销售额、中奖金额、Chance/Billete 热门号码与销售明细。
- **销售限额与单价**：可设每号最多卖多少张、Billete/Chance 单价（默认 1 美元 / 0.25 美元），保存后顾客端下单会按此价格计算。
- **开奖结算**：跳转到 `result.html` 看本店本期销售、赔付、利润及开奖结果。

---

## 五、总后台功能（简单易懂）

- **手动开奖**：选择日期与时间，填写 1/2/3 奖号码，点「发送开奖结果」；可单独「发送开奖时间」让各端倒计时一致。
- **店铺销售明细**：输入店号可查该店 Chance/Billete 销售明细。
- **全部店铺总销售**：汇总所有店铺的 Chance/Billete 总销量。
- **店铺对比**：选择时间段与 Top N，查看各店销售额与净利润柱状图。
- **最近 7 期结算记录**：在结算页（result）展示每期日期、奖号、销售/赔付/利润。
- **系统设置与状态**：可配置单笔/单日限额、未付订单超时时间等；显示 API、数据库、定时任务状态。

---

## 六、重要规则（简单说明）

- **订单 30 分钟**：下单后 30 分钟内未付款，订单自动取消；顾客端有倒计时，后端也有定时任务自动把超时单标为取消。
- **开奖时间**：默认每周三、周日 15:00（本地）；总后台可改并「发送开奖时间」同步到各端。
- **中奖与兑奖**：总后台发送开奖号码后，系统按 Billete 4 位 + Chance 2 位规则计算中奖；顾客中奖后拿二维码到店，老板扫码后点「确认兑奖」，该单标记为已兑奖，顾客端会显示「已兑奖」。
- **兑奖与店号**：平台只提供彩票能力，赚亏是老板与顾客的事，不防老板；老板可扫码、选单或输入订单号完成兑奖。二维码含店号，店号与当前店铺不一致时提示「此订单不是在此店买的」；详见第八节。

---

## 七、顾客订单付款后，数据存在哪里

- **真实数据在后端（数据库）**  
  顾客下单、老板确认收款、总后台开奖、老板兑奖，每一步都会更新**后端数据库**里的订单记录。订单号、号码、金额、状态（未付款/已付款/已开奖/已中奖）、中奖金额、是否已兑奖（`redeemed_at`）等，都存在后端 **Order 表** 里。

- **顾客端浏览器里只存“入口信息”**  
  顾客每次付款成功后，前端会在**本地（localStorage）**的已付款订单列表中**追加**一条摘要（订单号、金额、订单 hash、店铺号等），最多保留约 50 条，用来：
  - 在首页显示「已付款订单」列表（可多单）；
  - 点某一条时用订单号去**请求后端**拉取最新订单详情（状态、中奖金额、是否已兑奖等）。

- **总结**  
  | 内容           | 存在哪里                         |
  |----------------|----------------------------------|
  | 订单完整信息   | 后端数据库（唯一真实来源）       |
  | 顾客端看到的   | 每次打开详情页时从后端接口拉取   |
  | 本地存的       | 多笔已付款订单摘要（最多约 50 条），用于入口列表 |

## 八、兑奖与店号校验 — 具体说明

- **平台不防老板**  
  平台只提供彩票能力，具体赚与亏是老板和顾客的事。老板可以：
  - **扫**顾客的中奖二维码，弹出兑奖确认后点「确认兑奖」；
  - 在「等待兑奖」列表里**选中某单**，点该单的「完成兑奖」，再在弹窗中确认；
  - 在「等待兑奖」区**输入订单号**，点「完成兑奖」，若该单属于本店且已中奖未兑奖则弹出确认并完成兑奖。

- **二维码里有什么**  
  顾客中奖二维码内是一段 JSON，包含 `orderNumber`、`shopId`（店号）等。扫码后若二维码里的店号与当前登录店铺不一致，会提示「**此订单不是在此店买的**」，不进行兑奖。

- **输入订单号时的校验**  
  老板输入订单号点「完成兑奖」时，会先查单；若该订单的店铺不是当前店铺，同样提示「此订单不是在此店买的」。

- **顾客没保存好二维码怎么办？**  
  顾客在同一台手机、未清浏览器时，只要从首页「已付款订单」进入该单，完成页会再次显示同一中奖二维码，无需截图。若顾客换设备或找不到订单入口，可**记下订单号**，到店后由老板在「等待兑奖」处输入该订单号完成兑奖（前提是该单确属本店）。

## 九、项目结构（便于维护）

```
lottery-preview/          # 前端页面（顾客、老板、总后台、结算）
  index.html              # 顾客端
  merchant.html           # 老板端
  dashboard.html          # 总后台
  result.html             # 开奖结算页

lottery-system/backend/   # 后端 API（Node.js + NestJS）
  提供：下单、确认收款、查询订单、开奖、结算、兑奖、历史、店铺对比、健康检查等接口
  定时任务：每 1 分钟把超 30 分钟未付款的订单标为取消
```

## 十、使用前准备与上服务器测试

1. **启动后端**：在 `lottery-system/backend` 下执行 `npm install` 和 `npm run start`（端口可用环境变量 `PORT`，默认 3000）。
2. **打开前端**：用浏览器打开 `lottery-preview` 下的对应页面（或部署到静态服务器 / CDN）。
3. **配置地址**：各 HTML 里的 `API_BASE` 需指向你的后端地址；本地开发可用 `http://localhost:3000`，上服务器改为实际 API 域名（如 `https://api.xxx.com`）。
4. **后端 CORS**：当前已 `enableCors()` 允许任意来源；若上线要限制域名，可在 `lottery-system/backend/src/main.ts` 中配置 `origin`。
5. **顾客下单 IP**：顾客端下单时传的 `ipAddress` 为前端占位；若需真实 IP，需后端从请求头（如 `X-Forwarded-For`）读取。

## 十一、Cloudflare Tunnel 部署（生产环境）

使用 Cloudflare Tunnel 时，**公网域名**与**本地端口**对应关系（已确认可用）：

| 公网域名 | 本地服务 | 端口 |
|----------|----------|------|
| **www.casagrade.com** | 前端（lottery-preview 静态 + server.js） | **8080** |
| **api.casagrade.com** | 后端（lottery-system/backend NestJS） | **3000** |

- 前端启动：在 `lottery-preview` 下 `npm start`（默认监听 8080，可用 `PORT=8080`）。
- 后端启动：在 `lottery-system/backend` 下 `npm run start`（默认 3000，可用 `.env` 中 `PORT=3000`）。
- Tunnel 中配置两条 Public Hostname：`www` → `http://localhost:8080`，`api` → `http://localhost:3000`。
